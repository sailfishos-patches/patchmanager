Patchmanager Configuration Files

/etc/patchmanager2.conf

INI-style configuration file and state storage.

It currently consists of two sections: settings, and votes

settings

KeyTypedefaultDescription
appliedlist of strings, comma-separatedemptystores the list of "activated" patches
applyOnBootBooleanfalseWhether to activate Patches at boot, or show the {Patchmanager Documentation: Startup Dialog}{Startup Dialog}
bitnessMangleBooleanfalseConvert patch contents so they can apply on the local architecture (e.g. lib vs. lib64 path segments)
developerModeBooleanfalseWarning: deprecated. Pre-Patchmanager v3.2.7, this was used to store the settings for "Developer Mode". This has now been split into patchDevelMode and sfosVersionCheck
notifyOnSuccessBooleantrueWhether to show success messages in the UI
orderlist of strings, comma-separatedemptyThe order of the list of patches, bith in the UI as well as the order they will be applied/activated.
patchDevelModeBoolesnfalse
sfosVersionCheckInteger (Enum)0 (strict)Whether to relax the version checking in the PM GUI. The default will only allow Patches to be downloaded, or activated, whose compatibility field matches the currently running OS version exactly.
